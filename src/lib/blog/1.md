개발, 음악 등 저에 대한 모든 작업물과 기록을 한꺼번에 올릴 플랫폼을 가지고 싶었습니다.
기존 블로그 플랫폼은 입맛대로 UI/UX를 수정하는 일에 한계가 있었기 때문에 직접 홈페이지를 만들어 보았습니다.

Svelte-kit을 사용했습니다. 등장 초기에 잠깐 사용해봤고, 굉장히 개발이 편리하다고 느껴 선택했습니다. 그때보다 많이 발전한 게 느껴집니다만, 옛날과 다르게 지금은 AI의 도움을 받게 되면서 순수 svelte 기능만으로 아주 간단하게 구현할 수 있는 기능을 AI가 ts 스크립트를 짜주는 경우가 있어서 svelte의 장점이 희석되는 느낌이 있었습니다. 아무래도 학습 데이터에는 React 프로젝트 코드가 압도적으로 많을 테니 그럴 만도 하죠. 그래도 덕분에 프로젝트를 할 때는 컨텍스트 관리를 잘 해줘야겠다는 교훈을 얻었습니다. (이럴거면 그냥 React로 할 걸 하는 생각도 들었죠...)

어떤 UI/UX 요소, 인터랙션 등 있으면 좋겠다 싶은 것들을 죄다 때려 박았습니다. 그렇게 적용한 주요 기능들은 다음과 같습니다.

1. 다크모드 전환
2. 어디서든 보이고 움직일 수 있는 음악 플레이어
3. 블로그 글
    - 디렉토리 구조화. md 파일 기반
    - 포스트의 그림이 다크모드에서 색반전되도록
    - 글 리스트 아이템에 제목, 발행일 및 시간, 카테고리, 글자수가 표현되도록
    - 티스토리에서 이사 온 글들은 옆에 티스토리 아이콘을 눌러 찾아갈 수 있도록
    - 따라다니는 목차 버튼
    - 코드블럭 접기/펴기, 복사버튼
4. 프로젝트 쉽게 등록할 수 있도록 구조화
5. 툴팁 적극 활용
6. Github action, pages 배포
